Overview

Parallax will be a comprehensive, yet easy to use framework for creating and using compute clusters to execute computations in parallel across multiple processors in a single machine (SMP), among many machines in a cluster, grid or cloud. Parallax will be well suited for data parallel (SIMD) paradigm where a computation (Python function or standalone program) is evaluated with different (large) datasets independently with no communication among computation tasks (except for computation tasks sending Provisional/Intermediate Results or Transferring Files to the client).

Specifications

Parallax will be built upon asynchoro to provide multiprocessing base and we’ll be implementing our own system for load balancing and fault tolerance.

Similar libraries like parallel python and bloom may be used as inspiration.

Features

  1. Context aware node

Nodes will share the tasks based on the current availability of resources and reschedule as and when required

  1. Cluster management

We’ll be providing a browser based cluster dashboard using httpd module of python

  1. Dynamic addition of nodes and Fault tolerance

Nodes will be added or removed dynamically aiding to the fault tolerance mechanism and thus making the whole system robust

  1. Plug and play support

Cloud computing platforms like EC2, AWS or google compute engine can be used as compute nodes.
